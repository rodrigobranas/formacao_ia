# ADR-004: TopBar unitsSlot and Imperial Display Precision

## Status

Accepted

## Date

2025-06-25

## Context

`DESIGN.md` places the °C/°F segmented control in the sticky top bar **between search and geolocation actions**. The current `TopBar` component accepts only `searchSlot` and `actionsSlot`. The PRD left open questions on imperial label precision for pressure and near-zero precipitation.

## Decision

1. **TopBar layout**: Add an optional `unitsSlot: ReactNode` prop rendered between `{searchSlot}` and the `wx-actions` container. `WeatherDashboardPage` passes `<UnitToggle />` into `unitsSlot`. Mobile wrap order follows the design reference: brand → units → actions → search (full width).

2. **Pressure (inHg)**: Display **one decimal place** (e.g., `29.9 inHg`). This matches consumer weather apps and avoids misleading whole-number jumps for barometric readings.

3. **Precipitation (in)**: Display **two decimal places** (e.g., `0.04 in`) with no special `<0.01` threshold—values round normally; zero stays `0 in`.

## Alternatives Considered

### Alternative 1: Toggle inside actionsSlot

- **Description**: Render `UnitToggle` as the first child of `actionsSlot` alongside API pill and geolocation button.
- **Pros**: No TopBar API change.
- **Cons**: Groups units with actions rather than between search and geo; harder to achieve design-spec flex order on mobile.
- **Why rejected**: Design reference and `DESIGN.md` treat the toggle as its own top-bar element with distinct flex order.

### Alternative 2: TopBar owns UnitToggle internally

- **Description**: `TopBar` imports `UnitToggle` and accepts `unitSystem` + `onChange` props.
- **Pros**: Encapsulated header API.
- **Cons**: TopBar becomes state-aware; harder to test toggle in isolation; couples layout shell to feature component.
- **Why rejected**: Slot pattern keeps TopBar presentational and matches existing `searchSlot` / `actionsSlot` convention.

### Alternative 3: Whole-number pressure and zero-threshold precipitation

- **Description**: Round inHg to integers; show `<0.01 in` for trace amounts.
- **Pros**: Simpler display rules.
- **Cons**: Pressure loses meaningful precision; special threshold adds formatter branching for minimal UX gain.
- **Why rejected**: Stakeholder deferred precision to TechSpec; one decimal / two decimal is the industry norm.

## Consequences

### Positive

- TopBar remains a layout shell; toggle styling lives in `UnitToggle` + `.wx-units` CSS.
- Precipitation and pressure rules are deterministic for QA and unit tests.
- Mobile layout matches design reference without restructuring the page.

### Negative

- Slight API surface increase on `TopBar` (one optional slot).
- Two decimal precipitation may show `0.00 in` for truly zero values—acceptable per stakeholder choice.

### Risks

- **CSS order regression on mobile**: Mitigation: mirror design reference `order` values in `index.css` media query.
- **Visual inconsistency between whole-degree temp and decimal pressure**: Mitigation: expected—different measurement types use industry-standard precisions.

## Implementation Notes

- Add `.wx-units` styles from `docs/design/index.html` to `frontend/src/index.css`.
- `UnitToggle`: `role="group"`, `aria-label="Unidade de temperatura"`, buttons with `aria-pressed` for selected segment.
- Respect `prefers-reduced-motion`: no transition dependency for state change (existing global rule already disables transitions).
